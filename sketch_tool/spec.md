# Sketch Tool 技術規格書 (Specification)

## 1. 核心畫布與解析度規範
- **基準解析度**：768 x 1344 px (對應 9:16 比例)。
- **物理像素對準**：所有底圖載入、手繪筆跡儲存與最終合成影像均嚴格依循此解析度，以確保在 Demo 介面中達成 1:1 像素清晰度，避免縮放模糊或拉伸歪斜。

## 2. 圖層架構 (Layer Stack)
系統由下至上共分為四層主要顯示區域：
1. **背景底圖層 (bgCanvas)**：顯示載入的參考圖檔。載入時自動適應 768x1344 範圍並置中。
2. **手繪筆跡層 (inkCanvas)**：紀錄使用者的滑鼠/手寫筆跡。
3. **佈局參考層 (referenceLayer/lCtx)**：用於草稿預覽，不參與最終導出。
4. **互動文字層 (textOverlayContainer)**：基於 DOM 的互動式文字物件，支援拖曳移動、角落縮放與刪除。此層內容在導出時會即時投射回 Canvas 座標系進行渲染。

## 3. 工作區機制 (Workspace Concept)
為實現「專案化製作」與「無縫加載」，每個字都具備獨立的工作空間。
- **配置檔案**：`.sketch_config.json` (位於 `characters/[字]/` 下)。
- **設定格式**：
  ```json
  {
    "charName": "字名",
    "bgFilename": "底圖檔名.png",
    "elements": [
      {
        "text": "內容",
        "fontFamily": "字體名稱",
        "color": "HEX顏色",
        "fontSize": "像素大小",
        "left": "左距像素",
        "top": "頂距像素"
      }
    ]
  }
  ```

## 4. 資產導出與命名規範 (Export Assets)
每次執行「儲存工作區」或「存入字資料夾」時，系統會同步導出多個檔案：
- **`${charName}_${index}.png`**：完整合成影像（含底圖、筆跡與文字）。
- **`ink.png`** (固定名稱)：僅含文字的透明層影像。
- **`brush.png`** (固定名稱)：僅含手繪筆跡的影像，用於環境還原。

## 5. 自動化連動設計
- **一鍵置入**：置入文字按鈕會自動抓取當前開啟的工作區字名，無需重複輸入。
- **自動復原**：切換工作區時，會自動依序載入：`配置 JSON` -> `底圖檔名` -> `brush.png` (恢復筆跡) -> `互動文字元件`。
- **路徑連動**：所有存檔動作會自動鎖定為當前工作區對應的 `characters/` 目錄。

## 6. 伺服器通訊介面 (APIs)
- `POST /list`：取得 characters 目錄下的所有可用字資料夾。
- `POST /load`：讀取特定目錄下的 `.sketch_config.json`。
- `POST /save`：儲存圖檔與配置，支援 `isWorkspaceSync` 模式（僅更新設定，不增加圖檔編號）。

## 7. 字體支援
- 優先使用本機部署的 **「MasaFont (正風毛筆)」**。
- 提供毛筆草書、流暢行書、標準宋體等降級或切換選項。
